# Analysis Scheduler UI

## 目标

当前分析调度是全局单队列，虽然每盘棋可以选择 worker，但多个 worker 不能真正并行，多盘棋同时分析时也缺少清晰的全局队列展示。本次目标是把分析调度升级为按 worker lane 并发执行：不同 worker 可以同时分析，同一个 worker 串行执行；同时让 `析` 菜单展示当前棋局和所有 worker 的分析队列，并支持用户手动调整排队顺序。

## 决策

- 调度采用按 worker lane 并行模型。每个 worker 一条 lane，同一 worker 一次只分析一个节点，不同 worker 可以并行。
- 每盘棋固定使用它选择的 worker，不自动迁移到其他 worker。
- 后台整盘棋分析以“整盘棋”为普通队列单元，默认按加入顺序 FIFO 执行。
- 试下或手动落子产生的单节点分析进入该 worker 的最高优先级队列，排在所有整盘棋任务前面。
- 试下任务和手动插队都不打断当前正在分析的节点；当前节点结束后再执行队列最前面的任务。
- 用户可以在 `析` 展开菜单中把排队中的整盘棋手动插队到该 worker 普通队列最前。插队不超过试下任务。
- `析` 按钮本身显示当前棋局分析进度，例如 `12/180`；棋局列表只显示轻状态，例如“分析中”。
- `析` 展开菜单是全局分析状态入口，显示所有 worker lane、当前任务、试下任务、普通队列、棋局名、进度、模型和 visits。
- 停止分析只作用于当前棋局：移除该棋局在队列中的整盘任务；如果当前正在分析该棋局的节点，取消当前节点后不继续剩余节点；其他棋局不受影响。

## 架构

系统由调度器、worker pool、workspace 状态和 `析` 菜单四部分组成。调度器维护每个 worker 的 lane，每条 lane 有最高优先级单节点队列和普通整盘棋队列；worker pool 继续负责按 worker name 执行真实 KataGo 请求和取消请求；workspace 状态提供当前棋局分析进度；新增的调度状态快照提供全局 worker lane 信息给前端渲染。

### 调度器

调度器按 worker name 建 lane。后台分析任务按 game run 组织，一个 run 包含该棋局剩余待分析节点、generation、token、gameID、workerName、进度和状态。试下任务是独立单节点任务，放入同一 worker lane 的高优先级队列。

### 状态快照

后端在 workspace state 中暴露分析调度快照。快照至少包含每个 worker lane 的 workerName、当前任务、试下队列、普通队列、每个任务的 gameID、displayName、kind、analyzed、total、nodeID 和状态。前端只渲染后端状态，不重新推导调度语义。

### UI

`析` 菜单保留当前棋局的分析器选择、继续分析、重新分析、停止分析，同时增加全局 worker lane 区域。每个 worker lane 显示正在分析的任务、试下任务和普通队列。普通队列中的整盘棋任务提供“插队”按钮；当前任务、试下任务、已在普通队列第一位的任务不提供有效插队操作。

## 流程

1. 用户为棋局选择 worker 后点击继续分析或重新分析。
2. 后端创建该棋局的后台 run，并加入对应 worker lane 的普通队列。
3. lane 循环在 worker 空闲时取任务：先取最高优先级试下任务，再取普通队列第一盘棋的下一个节点。
4. 一个整盘棋 run 的节点全部完成后，该 run 完成；中途停止时该 run 失效并从队列移除。
5. 用户试下或手动落子时，当前节点分析任务加入对应 worker lane 的最高优先级队列；如果当前 worker 正在跑其他节点，则等待当前节点结束后立即执行。
6. 用户在 `析` 菜单点击插队时，该整盘棋 run 移到对应 worker lane 普通队列最前。
7. 分析结果仍通过现有事件写入 workspace，并按现有规则持久化主线分析文件。

## 验收标准

- 连接两个可用 worker 时，两盘棋分别选择不同 worker 并开始分析后，可以同时各自占用一个 worker 执行。
- 两盘棋选择同一个 worker 时，同一时间只分析一个节点，第二盘棋在该 worker 普通队列中等待。
- 同一 worker 上，后台整盘棋任务默认按加入顺序执行；用户在 `析` 菜单插队后，被插队棋局成为普通队列第一。
- 试下或手动落子的单节点分析排在该 worker 所有整盘棋任务前面，但不会中断当前正在跑的节点。
- 停止当前棋局分析后，该棋局后续节点不再执行；其他棋局的队列和正在分析的节点继续运行。
- `析` 按钮显示当前棋局进度；棋局列表仅显示轻状态，不展示完整队列细节。
- `析` 展开菜单展示所有 worker lane，包括当前任务、队列任务、试下任务、模型和 visits。
- 指定 worker 断开或不可用时，该 worker lane 的任务显示不可用状态，不自动迁移到其他 worker。

### 测试

- 后端调度测试覆盖不同 worker 并行、同 worker 串行、普通队列 FIFO、手动插队、试下优先级、停止当前棋局、worker 不可用不迁移。
- worker pool 相关测试继续覆盖按 name 执行、busy 等待和 cancel 消息。
- handler/state 测试覆盖调度状态快照包含 worker lane、当前任务、队列和进度。
- 前端组件测试覆盖 `析` 按钮进度、菜单展示所有 worker lane、插队按钮调用、棋局列表只显示轻状态。

## 范围之外

- 不做跨 worker 自动负载均衡。
- 不做暂停整个 worker 的 UI。
- 不做任务历史记录或分析耗时统计。
- 不记录每个分析结果当时使用的 model/visits 版本。
- 不新增独立分析任务页面。
