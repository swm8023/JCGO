# Unified UI Navigation

## 目标

JCGO 当前把本地棋局、设置、导入来源、链接导入、元萝卜浏览分别实现为标题栏以下覆盖、居中对话框或全屏页面，导致页面层级不统一，主标题栏在某些覆盖状态下仍暴露无效操作。本次统一应用外壳、页面边界和系统返回规则，让每个流程都能从视觉与交互上明确当前位置及下一步返回位置。

## 决策

- 所有页面始终保留同一条应用标题栏，内容只替换标题栏以下的工作区。
- 首页标题栏显示 `JCGO` 和首页操作：本地棋局、导入、设置，以及现有棋盘工具。
- 非首页页面把标题栏切换为上下文栏：左侧返回，中间当前页标题，右侧仅显示该页面真正可用的操作。首页的菜单、导入、设置和棋盘工具不会继续显示或可点击。
- 具有标题、滚动内容或多步流程的界面一律是工作区页面：本地棋局列表、导入来源、链接导入、元萝卜登录、元萝卜棋局浏览、设置。
- 仅单次选择或确认保留为弹层：分析操作菜单、元萝卜棋手/平台选择、删除确认。弹层始终阻止对下层页面的指针操作。
- 标题栏返回、浏览器返回、系统返回和 `Escape` 使用同一段应用历史，每次只退回一个层级。
- 导入流程维持可逐层返回：首页 -> 导入来源 -> 链接导入或元萝卜 -> 棋手/平台选择。选择棋局后回到棋盘首页；导入完成后的既有业务去向保持不变。

## 架构

`App` 维护唯一的页面层级状态并映射到浏览器 history。新增统一的应用外壳组件负责固定标题栏和工作区，页面组件只渲染工作区内容及其上下文操作。弹层由当前页面拥有，不改变主页面结构，只在历史中作为该页面的下一层记录。

## 流程

1. 首页打开一级工作区页面时，写入一条 history 记录，并以页面上下文标题栏替换首页标题栏。
2. 页面打开子页面或选择弹层时，再写入一条 history 记录。
3. 标题栏返回、`Escape`、浏览器或系统返回都消费一条记录，恢复上一个页面或弹层状态。
4. 页面覆盖期间，首页动作不在 DOM 可操作区域中；只有当前上下文栏和当前工作区接受交互。

## 验收标准

- 本地棋局、设置、导入来源、链接导入、元萝卜登录和元萝卜棋局浏览在桌面与移动布局中均保留统一标题栏。
- 非首页标题栏不显示或不响应首页专用按钮；不会出现可见但无效的主栏按钮。
- 上述六类界面不再使用居中对话框，均完整覆盖标题栏以下工作区并可独立滚动。
- 分析菜单、棋手/平台选择和删除确认仍为短操作弹层，且下层内容不能被点击。
- 从任意嵌套导入步骤连续触发系统返回、浏览器返回、标题栏返回或 `Escape`，每次都恰好回退一级，最终回到棋盘首页。
- 从本地棋局列表选择棋局后回到棋盘首页；现有导入成功后的业务去向不被本次改动改变。

### 测试

- 扩展 App 导航测试，覆盖每个一级页面的标题栏状态、逐层 history 回退、`Escape` 和系统返回。
- 扩展组件测试，验证工作区页面没有首页操作，弹层具备模态交互语义。
- 运行现有前端单元测试和生产构建；不在本次范围内新增后端测试。

## 范围之外

- 不改棋盘分析、Worker 配置字段或导入后端协议。
- 不重新设计棋局行、元萝卜数据内容或棋盘工具本身，只统一其所在页面的外壳和导航层级。
