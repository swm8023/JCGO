# YuanluoBo Account Import

> 由 scope skill 于 2026-07-08 生成

## 目标

JCGO 目前支持本地 SGF 导入和单条元萝卜复盘 URL 导入。这个功能要在加号导入入口中新增“元萝卜”入口，让用户通过元萝卜 App 扫码登录自己的账号，浏览账号下的历史棋局，并把选中的棋局导入到 JCGO。导入后的元萝卜棋局需要能在列表中识别为“已导入”，再次点击时直接打开本地棋局，不重复创建副本。

## 决策

- 登录方式只支持元萝卜 App 扫码登录，不做短信验证码登录。
- 元萝卜登录态保存到本地 `.data`，包括后续请求所需的 `token` 和 `uid`。
- 如果元萝卜接口返回 401 或“用户凭据丢失”，JCGO 清除本地元萝卜登录态并要求重新扫码。
- UI 入口放在现有加号导入流程中，新增“元萝卜”入口。
- 未登录时显示扫码登录；登录成功后显示账号、棋手、官方分类和棋局列表。
- 棋局分类使用元萝卜官方分类，按 `gameMode` 请求。
- 棋局列表每页显示 10 局，按对局时间倒序显示，最近的在前。
- 第一版不做时间范围筛选。
- 支持切换元萝卜账号下的棋手。
- 每条棋局显示是否已导入。
- 未导入棋局点击后导入并打开；已导入棋局点击后直接打开本地已有棋局。
- 重复判断只覆盖新功能之后从元萝卜账号列表导入的棋局。
- 不回溯识别此前通过“粘贴元萝卜链接”导入的棋局。
- 本地棋局记录需要保存来源标识：`source_platform = "yuanluobo"` 和 `source_id = <session_id>`。
- 元萝卜 token 不写入日志，不在前端错误消息中暴露。

## 架构

功能由后端元萝卜客户端、登录态存储、本地棋局来源标识、JSON-RPC 方法和前端导入 UI 组成。后端负责所有元萝卜接口请求、登录态读写、已导入判断和 SGF 导入；前端只负责展示扫码、分类、分页列表和点击动作。

### 后端元萝卜客户端

新增元萝卜客户端，封装以下官方接口：

- `GET /sso/permit/v1/qrcode`：创建扫码登录二维码。
- `GET /sso/permit/v1/qrcode/poll?key=...`：轮询扫码状态并取得 `token`、`uid`。
- `POST /r2/usercenter/v2/users/me/getOrAdd`：读取当前账号信息。
- `GET /r2/usercenter/v2/players?groupId=...`：读取账号下棋手。
- `POST /r2/chess/wq/sdr/v1/record/list`：按棋手、分类、分页读取棋局。
- `POST /r2/chess/wq/sdr/v3/record/detail`：复用现有单局详情转换 SGF。
- `POST /sso/v1/logout`：退出元萝卜登录。

所有需要登录的请求带元萝卜官方前端同等请求头：

```text
AUTH-TOKEN: <token>
AUTH-USERID: <uid>
SERVER-VERSION: 1.0.1
SOURCE: APP
CLIENT-TYPE: APP
AUTH-PRODUCT-NAME: SenseRobot-Go
Accept-Language: zh-CN
```

### 登录态存储

元萝卜登录态保存到 JCGO 数据目录下的本地文件中。该文件只保存元萝卜接口调用必需字段，退出登录或认证失败时删除。日志和 RPC 响应都不返回原始 token。

### 棋局来源标识

本地 `games` 记录增加来源字段，用于判断元萝卜列表项是否已导入。元萝卜账号列表导入成功时写入 `source_platform = "yuanluobo"` 和 `source_id = session_id`。列表接口根据这些字段把远端棋局标记为已导入，并返回对应本地 `gameId`。

### JSON-RPC

新增元萝卜相关 RPC：

- `yuanluobo.loginStart`：返回扫码二维码信息。
- `yuanluobo.loginPoll`：轮询扫码状态，登录成功时保存登录态。
- `yuanluobo.status`：返回当前是否已登录及基本账号信息。
- `yuanluobo.logout`：退出并清除本地登录态。
- `yuanluobo.players`：返回当前账号棋手列表。
- `yuanluobo.records`：返回指定棋手、分类和页码下的棋局列表。
- `yuanluobo.importRecord`：按 `session_id` 导入远端棋局；若已导入则直接返回本地棋局。

### 前端

导入入口新增“元萝卜”。进入后：

- 未登录时显示扫码二维码、扫码状态和刷新二维码动作。
- 登录后显示退出登录、棋手选择、官方分类 Tab、分页棋局列表。
- 棋局列表显示对局双方、时间、分类信息和“已导入”标记。
- 点击未导入棋局时调用导入 RPC，成功后打开该棋局。
- 点击已导入棋局时直接打开本地棋局。

## 流程

1. 用户点击加号并选择“元萝卜”。
2. 前端调用 `yuanluobo.status`。
3. 未登录时调用 `yuanluobo.loginStart`，显示二维码。
4. 前端每 3 秒调用 `yuanluobo.loginPoll`。
5. 扫码成功后后端保存登录态，并读取账号信息。
6. 前端读取棋手列表和当前分类第一页棋局。
7. 用户切换官方分类或页码时，前端调用 `yuanluobo.records`，每页请求 10 局。
8. 后端请求元萝卜 `record/list`，再用本地来源字段补充 `imported` 和 `gameId`。
9. 用户点击未导入棋局，后端拉取详情、转换 SGF、创建本地棋局、写入来源字段并返回打开所需状态。
10. 用户点击已导入棋局，前端直接选择返回的本地 `gameId`。
11. 用户退出元萝卜登录时，后端调用 logout 并删除本地登录态。

## 验收标准

- 加号导入入口中能看到“元萝卜”入口。
- 未登录进入元萝卜页面时能显示扫码登录二维码。
- 用户用元萝卜 App 扫码确认后，JCGO 能进入已登录状态。
- 重启 JCGO 后，如果元萝卜 token 仍有效，进入元萝卜页面不需要重新扫码。
- 元萝卜 token 失效或凭据丢失时，JCGO 清除本地登录态并提示重新扫码。
- 登录后能看到账号下棋手，并能切换棋手。
- 登录后能看到元萝卜官方分类 Tab。
- 每个分类下每页显示 10 局，最近棋局在前。
- 翻页或加载更多能看到更早棋局。
- 未导入棋局没有“已导入”标记，点击后导入并打开。
- 已导入棋局显示“已导入”标记，点击后直接打开本地棋局，不创建副本。
- 通过新元萝卜账号列表导入的棋局写入 `source_platform` 和 `source_id`。
- 此前通过粘贴 URL 导入的棋局不会被回溯标记为已导入。
- 退出元萝卜登录后，本地元萝卜登录态被清除。
- 日志和前端错误消息不泄露元萝卜 token。

### 测试

- 后端单元测试覆盖元萝卜扫码状态解析、认证头构造、登录态文件读写和认证失败清理。
- 后端单元测试覆盖 `games` 来源字段迁移、按来源查找已导入棋局、重复导入返回已有棋局。
- 后端单元测试覆盖元萝卜列表响应到前端记录模型的转换，包括 `imported` 和 `gameId`。
- 后端 RPC 测试覆盖 `yuanluobo.records` 未登录、已登录、凭据失效、分页参数和分类参数。
- 前端组件测试覆盖未登录扫码界面、已登录列表界面、分类切换、分页、已导入标记和点击行为。
- 不要求自动化测试真实元萝卜线上扫码；真实接口连通性用手动验收验证。

## 范围之外

- 不做短信验证码登录。
- 不做微信公众号、服务号或微信小号转发入口。
- 不做按 `player_id` 或 `session_id` 枚举他人棋局。
- 不做时间范围筛选。
- 不做批量导入。
- 不回溯标记旧的元萝卜 URL 导入棋局。
- 不把元萝卜登录态同步到其他设备。
