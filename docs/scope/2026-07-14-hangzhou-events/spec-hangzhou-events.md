# JCGO 杭州比赛入口

## 目标

JCGO 左上角的功能入口已经拥挤。将最左侧按钮改为唯一总入口，在应用内提供类似微信的底部 Tab 导航，整合现有的本地棋局、添加和设置功能，并新增可按月查看杭州市云比赛列表的入口。

## 决策

- 侧边栏主界面只保留最左侧的总入口按钮；现有“导入棋谱”和“设置”独立按钮收入总入口内。
- 总入口默认打开“本地棋局”，根页面底部显示四个 Tab：“本地棋局”、“添加”、“云比赛”、“设置”。
- Tab 样式参考微信的图标加文字底部导航，但继续使用 JCGO 现有的颜色、间距和控件语言。
- 四个 Tab 的根页面始终显示底部导航。进入“添加”下的链接导入、元萝卜等二级流程时隐藏底部导航，使用顶部返回按钮回到“添加”。
- 云比赛列表固定查询浙江省杭州市，不提供其他城市筛选。默认使用设备当前月份，用户可切换任意月份。
- 前端直接请求云比赛公开 HTTPS API，不增加 JCGO 后端代理或配置。
- 列表仅展示比赛名称、比赛日期、项目、报名费、已报人数和主办方，不展示接口返回的联系人电话等其他字段。
- 点击比赛在新标签页打开云比赛原始详情页 `https://m.yunbisai.com/signUp?eventid=<event_id>`。

## 架构

扩展现有 `AppHistoryLayer` 页面层级，将“本地棋局”、“添加”、“云比赛”和“设置”作为同级根页面，由共享的底部 Tab 切换。云比赛功能使用独立、聚焦的 API 模块负责请求参数、返回数据校验和上游字段映射，页面组件只管理月份、加载状态和展示。

## 流程

1. 用户点击左上角总入口，打开“本地棋局”根页面。
2. 用户通过底部 Tab 切换到“云比赛”。
3. 页面以 `province_name=浙江省`、`city_name=杭州市` 和选定的 `YYYY-MM` 月份请求公开列表接口，并将返回值映射为 JCGO 内部比赛类型。
4. 切换月份时发起新请求；如果前一个请求尚未结束，它不得覆盖新月份的结果。
5. 点击列表项使用 `noopener,noreferrer` 打开对应的云比赛详情页。

## 验收标准

- JCGO 主界面左上角只用一个总入口承载本地棋局、添加、云比赛和设置。
- 根页面在手机和桌面宽度下都有可键盘操作、可见焦点、图标与中文标签完整的底部 Tab 导航。
- 底部 Tab 的选中状态与当前根页面一致；浏览器返回键和顶部返回按钮不会将用户困在页面层级中。
- “本地棋局”、“添加”和“设置”保持现有功能和内容。
- “云比赛”默认显示当前月杭州市的比赛，切换月份后只显示新月份的杭州市比赛。
- 比赛列表字段、费用和人数有明确标签；免费比赛显示“免费”，跨天比赛显示完整日期范围。
- 点击比赛可打开与该 `event_id` 对应的云比赛原始详情页。
- 加载期间有明确状态；当月无比赛时显示空状态；跨域、网络或上游数据失效时显示可重试的错误状态，不影响 JCGO 其他功能。

### 测试

- API 模块单元测试覆盖杭州与月份查询参数、正常字段映射、免费费用和非法返回值。
- 页面组件测试覆盖默认月份、月份切换、加载、空、错误与重试状态，以及比赛详情链接。
- 导航测试覆盖总入口、四个根 Tab、二级添加流程和浏览器返回行为。
- 现有前端测试和生产构建必须通过。不在自动化测试中依赖云比赛线上接口，使用可控的 `fetch` 返回值验证前端行为。

## 范围之外

- 不在 JCGO 内实现比赛报名、支付、登录或详细规程页。
- 不提供杭州市以外的地区筛选，不增加项目类型、报名费或状态筛选。
- 不保存、缓存或后台同步云比赛数据。
- 不修改 JCGO Go 后端、部署配置或访问令牌机制。
